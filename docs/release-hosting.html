Release hosting

The startup flow uses GitHub Releases for versioned storage and GitHub Pages for a stable static read plane compatible with current spio clients.

Model

Large binaries should not be committed to this repository. Instead, a release-root bundle is produced outside the site source tree, uploaded to GitHub Releases, then extracted into the Pages deployment artifact.

Build CLI binaries spio and styio are built for each supported platform.
Stage Static root styio-platform writes tools/, channel pointers, and sha256 files.
Store GitHub Release The bundle is attached to a release tag for auditability.
Serve GitHub Pages Pages exposes the exact paths expected by installers.

Create a release-root bundle

Use styio-platform's publisher to place each binary into a temporary registry root. Example: 

release_root="$(mktemp -d)"

python3 /path/to/styio-platform/scripts/publish-spio-tool-release.py \
  --registry-root "$release_root" \
  --tool spio \
  --binary /path/to/spio \
  --binary-name spio \
  --version 0.1.0-alpha \
  --platform darwin-aarch64 \
  --install-script /path/to/styio-spio/scripts/install-spio.sh

python3 /path/to/styio-platform/scripts/publish-spio-tool-release.py \
  --registry-root "$release_root" \
  --tool styio \
  --binary /path/to/styio \
  --binary-name styio \
  --version 0.0.1 \
  --platform darwin-aarch64 \
  --release-target styio-macos-cli \
  --channel stable

scripts/validate-release-root.sh "$release_root"
tar -C "$release_root" -czf styio-release-root-0.1.0-alpha.tar.gz tools

Repeat the publish command for each supported platform before archiving the root.

Upload to GitHub Releases

gh release create release-root-0.1.0-alpha \
  styio-release-root-0.1.0-alpha.tar.gz \
  --repo eBioRing/styio.io \
  --title "Styio release root 0.1.0-alpha" \
  --notes "Static release root for spio 0.1.0-alpha and styio 0.0.1"

Deploy through Pages

Run the Pages workflow manually with the release bundle inputs: 

gh workflow run pages.yml \
  --repo eBioRing/styio.io \
  -f release_bundle_repo=eBioRing/styio.io \
  -f release_bundle_tag=release-root-0.1.0-alpha \
  -f release_bundle_asset=styio-release-root-0.1.0-alpha.tar.gz

The workflow builds the static site, downloads the bundle, extracts it into _site/, validates the release root, and deploys the combined artifact to GitHub Pages.

Policy

  • Do not commit files under tools/*/releases/.
  • Release-root bundles are immutable once announced publicly.
  • Every binary must have a matching .sha256 file.
  • Channel pointers must be plain text version files for shell compatibility.
  • GitHub Releases are the audit source; Pages is the static read plane.
