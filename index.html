Install Styio from a versioned release root.

Styio uses spio to install the compiler, verify release checksums, and keep the local toolchain pinned to published CLI builds.

Install Styio Publish Releases
terminal 
curl -fsSL https://styio.io/tools/spio/install-spio.sh | sh -s -- --base-url https://styio.io
spio install styio@latest --prebuilt-only
styio --version

Static read plane

Pages serves installer scripts, channel pointers, checksums, and release binaries through stable paths.

Release assets stay outside Git

Operators upload a release-root bundle to GitHub Releases. The Pages workflow extracts it into the deployment artifact.

Client-compatible today

The layout matches current spio lookup rules for macOS CLI, Linux glibc CLI, and Linux musl CLI builds.

Current alpha surface

The first public flow is intentionally narrow: install spio, use it to install the prebuilt Styio compiler, then verify the compiler.

Supported first targets

  • OKmacOS arm64 CLI: styio-macos-cli/darwin-aarch64
  • OKLinux glibc arm64 CLI: styio-linux/linux-aarch64
  • OKLinux musl arm64 CLI: styio-linux/linux-musl-aarch64

Distribution contract

  • OKShell installer uses curl and checksum verification.
  • OKspio install styio@latest reads static channel pointers.
  • OKLarge release bundles are generated outside repository history.

Release pipeline

styio-platform owns artifact publication. This repository owns the public static surface and Pages deployment.

1 Build Produce spio and styio binaries for supported platforms.
2 Stage Use the platform publisher to create a static release-root tree.
3 Archive Upload the release-root bundle as a GitHub Release asset.
4 Deploy Run the Pages workflow to expose the release root and docs.
